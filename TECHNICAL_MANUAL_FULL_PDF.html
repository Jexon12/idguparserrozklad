Schedule Viewer — Розширений технічний посібник

Версія 1.0 · 30.04.2026
Щоб отримати PDF: Ctrl+P → Save as PDF → увімкніть Background graphics.

1. Призначення

Система автоматизує роботу з розкладом і сесіями: перегляд, аналіз, контроль конфліктів, експорт і адміністрування даних.

2. Основні сторінки

  • index.html — головний desktop-інтерфейс.
  • index2.html — mobile-інтерфейс.
  • session.html — перегляд сесії з фільтрами.
  • session-admin.html — завантаження DOCX і керування сесіями.
  • session-constructor.html — ручне редагування + конфлікти.
  • session-prep.html — таблиця предмет/група/викладачі.
  • builder.html, smart.html, course-live.html — аналітика і оперативний контроль.

3. API та backend

Ключові маршрути

  • GET /api/health
  • GET /api/session, POST /api/session
  • GET/POST /api/times, GET/POST /api/links
  • POST /api/report/start, GET /api/report/status, GET /api/report/download
  • GET /api/monitor, POST /api/monitor/log
  • GET /api/audit, GET /api/versions
  • GET /api/<action> — proxy до зовнішнього API розкладу.

Кешування і dedupe

  • In-memory cache + optional Redis/KV.
  • TTL: default 5 хв, schedule 3 хв.
  • Dedupe одночасних запитів (in-flight).

Rate limit

  • Admin POST: 20/хв/IP.
  • Proxy: 120/хв/IP.

4. Функції по сторінках (операційно)

Session Admin

  1. Ввести пароль, назву сесії, форму навчання.
  2. Додати DOCX.
  3. Натиснути “Розпарсити”.
  4. Натиснути “Завантажити в API”.

Session Constructor

  • Поля: предмет, група, викладачі, тип контролю, дата, час (іспит), аудиторія.
  • Real-time конфлікти: група/викладач/аудиторія.
  • Експорт в Excel, upload в API.

Course Day

  • Оновлення тільки вручну кнопкою Оновити зараз.
  • Heatmap, room conflicts, teacher load/windows, data quality, compare dates, presets.

5. Локальний запуск і тести

npm install
npm run build:css
npm start

npm test -- --runInBand
npm test -- --runInBand --detectOpenHandles
npm run lint:encoding

6. Troubleshooting

  • Немає даних: перевірити фільтри + дату + натиснути “Оновити зараз”.
  • 401/403 admin: перевірити ADMIN_PASSWORD і правильний пароль у формі.
  • Старий UI: Ctrl+F5, перевірити кеш браузера/SW.
