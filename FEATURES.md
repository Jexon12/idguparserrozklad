# Функции приложения `schedule-viewer`

## 1) Основной просмотр расписания (`index.html`)
- Поиск и отображение расписания по:
  - группе;
  - преподавателю;
  - аудитории.
- Просмотр расписания по диапазону дат.
- Переключение режимов: неделя/список.
- Быстрые фильтры:
  - офлайн/онлайн;
  - по дисциплине;
  - по типу занятия.
- Показ текущей и следующей пары.
- Сохранение пользовательских настроек (включая тему).
- Поддержка тёмной/светлой темы.
- Экспорт отчётов по преподавателю в Excel (через backend job + прогресс).
- Скан занятости аудиторий с прогрессом и экспортом.
- Заметки/вспомогательные пользовательские данные.

## 2) Мобильная версия (`index2.html`)
- Упрощённый мобильный UI для быстрого доступа.
- Переключение между desktop/mobile.
- Карточки “сегодня/завтра”.
- Кнопки быстрых действий (поиск, календарь, очистка данных, настройки).
- Отображение расписания с ключевой информацией для телефона.

## 3) Страница сессии (`session.html`)
- Загрузка данных сессий из API.
- Фильтры:
  - по сессии (term);
  - по форме обучения;
  - по группе;
  - по преподавателю (мультивыбор);
  - по дисциплине;
  - по типу контроля;
  - по дате.
- Быстрые чипы групп.
- Таблица сессии + счётчик найденных записей.
- Экспорт отфильтрованного результата в XLSX.

## 4) Админка сессий (`session-admin.html`)
- Загрузка DOCX-файлов сессии.
- Парсинг нескольких файлов в один набор данных.
- Выбор названия сессии (term) и формы обучения.
- Защищённая отправка в API по паролю админа.
- Объединение данных в существующую сессию (без потери уже загруженных записей).
- Управление сессиями:
  - обновление списка;
  - переименование;
  - перемещение в корзину;
  - восстановление;
  - удаление навсегда.
- Экспорт backup JSON.
- История действий (операции над сессиями).

## 5) Smart-аналитика (`smart.html`)
- Smart Day:
  - текущая/следующая пара;
  - аналитические блоки.
- Рейтинг удобства расписания (0–100) на основе метрик.
- Теплокарта нагрузки (день × пара).
- What-if симулятор (оценка сценариев изменений).
- Live board для оперативного просмотра изменений/событий.

## 6) Week Builder / конструктор (`builder.html`)
- Сбор расписания факультета за неделю.
- Анализ:
  - число занятий;
  - активные дни;
  - конфликтные слоты;
  - дубликаты;
  - источники.
- Вывод по группам в табличном формате.
- Режим оптимизации и рекомендации.
- Экспорт результатов/матриц в Excel.

## 7) Новая страница подготовки сессии (`session-prep.html`)
- Формирование таблицы “Предмет | Группа | Преподаватели”.
- Выбор:
  - факультет;
  - форма обучения;
  - курсы (чекбоксы);
  - период дат.
- Автозагрузка групп выбранных курсов (чекбоксы).
- Парсинг расписания для каждой выбранной группы.
- Нормализация дисциплин и преподавателей.
- Удаление дублей/пустых/технических значений.
- Поиск, фильтр по группе, фильтр по преподавателям (чекбоксы).
- Копирование таблицы в буфер.
- Экспорт в Excel.
- Индикатор загрузки и сообщения об ошибках.

## 8) Серверные API-функции (`api/index.js`)
- Proxy к API расписания вуза (`/api/*`).
- Кеширование “тяжёлых” запросов + дедупликация in-flight.
- Job-очередь для отчётов (чтобы не блокировать UI).
- Эндпоинты отчётов:
  - `/api/report/start`;
  - `/api/report/status`;
  - `/api/report/download`.
- Эндпоинты сессий:
  - `GET/POST /api/session`.
- Эндпоинты админ-данных:
  - `GET/POST /api/times`;
  - `GET/POST /api/links`.
- Monitoring:
  - `GET /api/monitor`;
  - `POST /api/monitor/log`.
- Audit/version:
  - `GET /api/audit`;
  - `GET /api/versions`.
- Инвалидация кеша:
  - `POST /api/cache/invalidate`.
- Rate limit для чувствительных операций.

## 9) UX/производительность/надёжность
- Частичная загрузка и ленивое подключение модулей (в ряде сценариев).
- Web Worker для тяжёлого скана занятости.
- Защита от дублирующихся запросов.
- Сервис-воркер (кеш статики/офлайн-поведение — в рамках текущей реализации).
- Smoke/UI/API тесты для контроля регрессий.
- Проверка кодировки (UTF-8/mojibake detector) для CI.

## 10) Тестирование
- `tests/smoke-pages.test.js`:
  - доступность страниц и health endpoint.
- `tests/ui-links-buttons.test.js`:
  - проверка внутренних ссылок/ресурсов;
  - проверка привязки кнопок и обработчиков.
- `tests/api.test.js`:
  - ключевые API-роуты;
  - auth-сценарии;
  - отчёты (успех/ошибка).

## 11) CI
- GitHub Actions workflow:
  - установка зависимостей;
  - проверка кодировки (`npm run lint:encoding`);
  - запуск тестов (`npm test -- --runInBand`).
