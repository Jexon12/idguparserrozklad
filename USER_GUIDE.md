# Schedule Viewer — Повна інструкція користувача

Оновлено: 30.04.2026

## 1. Швидкий старт

1. Відкрийте головну сторінку: `index.html` (desktop) або `index2.html` (mobile).
2. Оберіть режим:
   - `🎓 Група` — для розкладу груп.
   - `👨‍🏫 Викладач` — для розкладу викладача.
   - `🏫 Ауд.` — для скану зайнятості аудиторій.
3. Оберіть факультет, форму, курс, групу (або кафедру/викладача).
4. Вкажіть діапазон дат.
5. Натисніть `+ Додати`.

## 2. Головна сторінка (Desktop/Mobile)

### 2.1 Пошук і додавання розкладу
- Кнопки:
  - `+ Додати` — додає вибрану групу/викладача.
  - `⭐` — додати в обране.
- Щоб видалити сутність з порівняння — натисніть `×` на її картці.

### 2.2 Фільтри
- `Усі / Офлайн / Онлайн` — фільтр формату занять.
- Фільтр по дисципліні та типу заняття.
- Перемикач відображення: `Картки` або `Таблиця`.

### 2.3 Поточна/наступна пара
- Верхній банер показує активну пару, прогрес та час до кінця.
- Віджет “Сьогодні/Завтра” (mobile) показує кількість пар.

### 2.4 Експорт та інтеграції
- `📊 Звіт` — генерація Excel-звіту (по викладачу).
- `📆 Calendar` — додавання події в календар.
- `🔗 Share` — поділитись поточним станом.

### 2.5 Обране
- Блок `⭐ Обране`:
  - швидке перемикання;
  - шеринґ набору обраних;
  - відновлення обраного через URL.

### 2.6 Налаштування та очистка
- `⚙️` — модальне вікно налаштувань.
- `🗑 Очистити` — очистка локальних даних.
- `🌙/☀️` — темна/світла тема.

## 3. Сторінка “Сесія” (`session.html`)

### 3.1 Для чого
Перегляд і фільтрація екзаменаційної сесії у таблиці.

### 3.2 Що можна фільтрувати
- Назва сесії (term).
- Форма навчання.
- Група.
- Викладач(і).
- Дисципліна.
- Тип контролю.
- Дата.

### 3.3 Дії
- `Експорт XLSX` — експорт саме відфільтрованих рядків.
- Пошук зверху таблиці — швидкий текстовий фільтр.

## 4. Адмінка сесій (`session-admin.html`)

### 4.1 Що робить
Завантажує DOCX-файли сесій, парсить та відправляє в API.

### 4.2 Як користуватись
1. Введіть адмін-пароль.
2. Вкажіть назву сесії (наприклад, `Зимова 2025-26`).
3. Оберіть форму навчання.
4. Додайте 1+ DOCX файлів.
5. Натисніть `1) Розпарсити файли`.
6. Перевірте попередній результат.
7. Натисніть `2) Завантажити в API`.

### 4.3 Управління сесіями
- Оновити список сесій.
- Перейменувати.
- Архівувати/відновити.
- Видалити назавжди.
- Експорт JSON backup.

## 5. Конструктор сесії (`session-constructor.html`)

### 5.1 Для чого
Ручне редагування сесії з автоматичними підказками та перевірками.

### 5.2 Основні елементи
- Таблиця полів:
  - `Предмет`
  - `Група`
  - `Викладачі`
  - `Форма контролю` (`залік`, `іспит`, `захист`)
  - `Дата`
  - `Час (тільки іспит)`
  - `Аудиторія`
- Кнопки:
  - `+ Додати свій предмет`
  - `Автодати (залік/іспит)`
  - `Smart-підказки`
  - `Завантажити в API`
  - `Експорт Excel`

### 5.3 Перевірки в реальному часі
- Конфлікти групи (два іспити одночасно).
- Конфлікти викладача (дві події одночасно).
- Конфлікти аудиторії (два іспити в одній аудиторії в один час).
- Відсутні поля (дата/час/аудиторія/викладач).

## 6. Підготовка до сесії (`session-prep.html`)

### 6.1 Що робить
Автоматично формує зведену таблицю:
`№ | Предмет | Група | Викладачі`.

### 6.2 Як користуватись
1. Оберіть факультет, форму навчання, курс(и), період.
2. За потреби відфільтруйте групи/викладачів чекбоксами.
3. Натисніть `Сформувати таблицю`.
4. Використовуйте пошук і фільтри.
5. Натисніть `Експорт в Excel` або `Скопіювати таблицю`.

## 7. Smart Day (`smart.html`)

### 7.1 Можливості
- Рейтинг зручності розкладу (0–100).
- Теплокарта навантаження.
- What-if симулятор.
- Live board змін на день.

## 8. Week Builder (`builder.html`)

### 8.1 Для чого
Збір/аналіз розкладу по факультету за тиждень і оптимізаційні сценарії.

### 8.2 Режими
- Базовий збір поточного розкладу.
- Допоміжний аналіз (без змін).
- Новий оптимізований розклад.

### 8.3 Вивід
- Таблиці по групах.
- Конфлікти/вікна/дублі.
- Експорт у Excel.

## 9. Course Day (`course-live.html`)

### 9.1 Важливо
Автозавантаження вимкнене. Дані тягнуться тільки після натискання `🔄 Оновити зараз`.

### 9.2 Основні кроки
1. Оберіть дату.
2. Оберіть факультети, форми, курси (чекбокси).
3. Натисніть `🔄 Оновити зараз`.
4. За потреби:
   - `Знайти вікна` (спільні вікна),
   - `Порівняти з датою`,
   - `Тільки зміни`,
   - `Експорт поточного`.

### 9.3 Додаткові блоки
- Heatmap.
- Конфлікти аудиторій.
- Навантаження викладачів.
- Окна викладачів.
- Панель якості даних.
- Live Board.
- Переходи між корпусами.
- Журнал дій.

### 9.4 Пресети
- `Зберегти пресет` — зберігає поточні чекбокси.
- Вибір пресета зі списку — застосування.
- `Видалити пресет` — видалення.

## 10. Типові задачі “куди натискати”

### 10.1 Хочу просто переглянути розклад групи
`index.html` → `Група` → факультет/форма/курс/група → дати → `+ Додати`.

### 10.2 Хочу розклад викладача
`index.html` → `Викладач` → факультет/кафедра/викладач → дати → `+ Додати`.

### 10.3 Хочу знайти вільні аудиторії зараз
`index.html`/`index2.html` → `Ауд.` → дата → `Скан` → `Вільні зараз`.

### 10.4 Хочу підготувати дані сесії з DOCX
`session-admin.html` → пароль → назва сесії → файли → `Розпарсити` → `Завантажити в API`.

### 10.5 Хочу ручне редагування сесії
`session-constructor.html` → редагуйте таблицю → перевірте конфлікти → `Завантажити в API` або `Експорт Excel`.

### 10.6 Хочу список “предмет-група-викладачі”
`session-prep.html` → вибір факультету/форми/курсів/дат → `Сформувати таблицю` → `Експорт`.

### 10.7 Хочу аналіз по курсам на день
`course-live.html` → фільтри → `Оновити зараз` → дивіться heatmap/конфлікти/вікна.

## 11. Якщо щось “не працює”

1. Зробіть `Ctrl+F5`.
2. Перевірте правильність дат і фільтрів.
3. Якщо admin-операція: перевірте пароль.
4. Перевірте `session` чи `course-live`, чи натиснуто кнопку оновлення.
5. Для production: перевірити `/api/health`.

## 12. Формат PDF

Рекомендовано:
1. Відкрийте файл `USER_GUIDE_PDF.html`.
2. Натисніть `Ctrl+P`.
3. Оберіть `Save as PDF`.
4. Увімкніть “Background graphics” для коректного вигляду.
