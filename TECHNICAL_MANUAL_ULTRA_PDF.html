1. Призначення системи

Платформа для керування розкладом, підготовки сесій, контролю конфліктів, експорту і операційної роботи деканату.

2. Карта сторінок

  • index.html, index2.html
  • session.html, session-admin.html, session-constructor.html, session-prep.html
  • smart.html, builder.html, course-live.html

3. Архітектура

  • Frontend: HTML + JS (Vue/vanilla).
  • Backend: api/index.js (serverless).
  • Storage: KV/Redis, fallback db.json.
  • Session fallback: data/session-2025-26.json.

4. API-контур

Критичні ендпоінти

  • GET /api/health
  • GET/POST /api/session
  • GET/POST /api/times
  • GET/POST /api/links
  • POST /api/report/start
  • GET /api/report/status
  • GET /api/report/download
  • GET /api/monitor, POST /api/monitor/log
  • GET /api/audit, GET /api/versions
  • GET /api/<action> (proxy до розкладу)

Кеш + захист

  • TTL: 5 хв default, 3 хв schedule.
  • In-flight dedupe на backend.
  • Rate-limit: admin 20/min, proxy 120/min.

5. Операційні сценарії

Session Admin

  1. Пароль → Назва сесії → Форма → DOCX.
  2. Розпарсити → перевірити preview.
  3. Завантажити в API.

Session Constructor

  • Редагування рядків вручну.
  • Контроль залік/іспит/захист.
  • Час тільки для іспиту.
  • Автодати + smart-підказки + конфлікти.

Course Live

  • Без автозавантаження: тільки кнопка Оновити зараз.
  • Compare date, only changes, heatmap, room conflicts, quality panel.

6. Конфлікти і правила

  • Група: не можна 2 іспити в один date+time.
  • Викладач: не можна 2 іспити в один date+time.
  • Аудиторія: не можна 2 іспити одночасно.

7. Запуск і тести

npm install
npm run build:css

npm test -- --runInBand
npm test -- --runInBand --detectOpenHandles
npm run lint:encoding

8. Troubleshooting

  • Старий UI: Ctrl+F5.
  • 401/403 admin: перевірити ADMIN_PASSWORD.
  • Порожні дані: перевірити фільтри/дату/натискання update.
